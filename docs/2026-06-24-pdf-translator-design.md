# PDF 双语翻译阅读器 — 设计文档

- 日期：2026-06-24
- 状态：设计已确认，待写实现计划
- 项目目录：`C:\APP\claude_sessions_space\pdf_translator\`

## 1. 目标

一个**本地桌面应用**，像「知云翻译」那样阅读英文 PDF，但翻译引擎可接入**用户自己的 API**，并支持多种大模型与传统翻译服务。核心是：边读 PDF 边即时获得高质量中文翻译，单词查词典、整页双语对照/原位替换，且尽量省钱（缓存）、省眼（护眼主题）、并打通「读论文 → 背单词」。

面向用户：读英文论文/外刊的科研人员（本人）。所有引擎国内直连，不依赖代理。

## 2. 核心功能

1. **基础 PDF 阅读**：打开、翻页、跳页、缩放、适应宽度、全文搜索
2. **划词翻译（两种内容模式，自动判断；交互见 §5.4）**
   - 选中**单个单词** → **词典卡片**：音标、词性、各词性释义、固定搭配/常用短语、例句、🔊 朗读、➕ 加入生词本
   - 选中**短语/句子** → **译文**（大模型逐字流式；传统引擎整段返回）+ 复制按钮
   - 触发：**选中后按键/点小图标**才翻译（不自动，避免误触）
   - 显示：**浮窗为主，可「📌 钉住」转为常驻侧栏**
3. **整页/整篇翻译（两种视图 + 两种范围）**
   - 视图：**双栏对照**（左原文/右译文）⇄ **原位替换**（原版面盖成中文），工具栏切换
   - 范围：默认**「翻译当前页」**；另有**「翻译整篇」**（进度条 + 翻译前 token/费用预估并确认）
4. **多翻译引擎可切换**（用户填自己的 key）
5. **多主题可切换**（含暗/浅）
6. **翻译缓存**：省 token、秒出；设置里查看大小 + 一键清理
7. **生词本 + 导出 Anki**：查过的单词可收藏，导出为 Anki 可导入格式（间隔重复背诵）
8. **术语表（glossary）**：自定义术语固定译法，注入提示词保证全文一致
9. **单词朗读**：词典卡片 🔊，系统 TTS，离线免费

## 3. 技术栈

| 部分 | 选型 | 理由 |
|------|------|------|
| GUI | Python + PySide6 (Qt6) | 成熟、打包方便；翻译走 QThread 后台线程，UI 不卡 |
| PDF 引擎 | PyMuPDF (fitz) | 渲染、取词坐标、全文搜索、原位擦写文本 |
| HTTP/流式 | httpx | 流式输出（划词打字机效果） |
| 缓存/生词本 | SQLite | 本地存翻译结果、生词 |
| 离线词典 | **ECDICT 精简版**（SQLite，数十 MB） | 免费开源英汉词典，含音标/词性/释义；完整版 700MB+ 太大，用精简版 |
| 朗读 TTS | 系统 TTS（Windows SAPI，如 pyttsx3） | 离线、免费 |
| API key 存储 | 系统 keyring（凭据管理器） | 不明文存盘 |
| 主题 | Qt QSS | 颜色抽成变量，每主题一份配色 |

## 5. 架构 — 组件划分

各组件单一职责、通过明确接口通信，可独立测试。翻译/网络一律在后台线程执行，通过信号回传 UI。

### 5.1 `PdfView`（左栏，阅读 + 取词）
- PyMuPDF 渲染页面为图像；支持缩放、翻页、跳页、适应宽度、全文搜索（高亮命中）
- 鼠标框选取文字 + 坐标（`get_text("words")` / `get_textbox`）；双击=选词，拖拽=选短语
- 选中后浮现「翻译」小图标 / 监听快捷键 → 发出「翻译此选区」事件
- **扫描件检测**：若页面无文字层（图片型 PDF），提示「此 PDF 无可提取文字（疑似扫描件），暂不支持，OCR 后续支持」
- **原位替换视图**：调 `InplaceRenderer`（§5.9）

### 5.2 `TextPreprocessor`（文本预处理 / 重排）★质量命门
整页翻译前清洗 PDF 文本：
- 合并被换行打断的句子成完整段落
- 去除行尾连字符断词（`evalu-\nation` → `evaluation`）
- 剔除页眉、页脚、页码、栏外噪声
- 处理**双栏论文的阅读顺序**（按块坐标判断左右栏，重排为正确顺序）
- 输出干净的段落列表给翻译层

### 5.3 `TranslationPane`（右栏）
- 双栏对照视图下，接收整页译文，按原文段落顺序排版，与左栏滚动联动

### 5.4 `WordCard / TransPopup`（划词浮窗，交互 1C + 2B）
- **触发**：选中后按快捷键或点小图标才翻译（不自动）
- **判断**单词 vs 短语：单词→词典卡片；短语→译文（流式追加）
- 词典卡片含：音标、词性、释义、固定搭配、例句、🔊 朗读、➕ 加入生词本
- 短语译文含：复制按钮
- **显示**：默认浮窗跟随选区；带「📌 钉住」→ 转右侧常驻侧栏，后续选中刷新到侧栏

### 5.5 `Translator`（翻译引擎抽象层）
统一接口，所有引擎实现，新增引擎不动主程序：

```
class Translator:
    def translate(self, text, target="zh") -> str
    def translate_stream(self, text, target="zh") -> iter   # 大模型支持；传统引擎退化为一次性
    def lookup_word(self, word) -> WordEntry | None          # 大模型补固定搭配/例句
```

支持引擎（**全部国内直连，无需代理**）：

| 家族 | 引擎 | 接口 | 默认模型 |
|------|------|------|----------|
| 大模型 | DeepSeek | `https://api.deepseek.com` | `deepseek-chat` |
| 大模型 | 智谱 GLM | `https://open.bigmodel.cn/api/paas/v4` | `glm-4-flash` / `glm-4` |
| 大模型 | MiniMax | `https://api.minimax.chat/v1` | `MiniMax-Text-01` |
| 大模型 | 通义千问 Qwen | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` / `qwen-turbo` |
| 大模型 | Kimi (Moonshot) | `https://api.moonshot.cn/v1` | `moonshot-v1-8k` |
| 大模型 | 豆包 Doubao (火山) | `https://ark.cn-beijing.volces.com/api/v3` | `doubao-pro` |
| 大模型 | 🌟 自定义(OpenAI 兼容) | 用户自填 base_url + key | 用户自填模型名 |
| 传统 | 有道云 | 有道智云 API，appKey+appSecret 签名(SHA256) | 直连，免费额度 |

- 大模型翻译用 chat 接口，系统提示词默认「专业学术翻译，术语准确，只输出译文」，可自定义；**注入术语表**（§5.8）
- 「自定义(OpenAI 兼容)」=万能入口，可接任意兼容服务（含本地部署开源模型）

### 5.6 `TranslateQueue`（批量调度，成本/稳定性）
- 整篇/整页批量翻译时：**限制并发数**（如 3~5）、**失败指数退避自动重试**、汇报进度
- 「翻译整篇」前按字符数估算 token/费用，弹确认框

### 5.7 `Dictionary`（单词词典，方案 A）
- 基础信息查本地 **ECDICT 精简版**（音标/词性/释义），离线秒出、0 成本
- 固定搭配/例句调当前大模型 `lookup_word` 补充（传统引擎时仅显示 ECDICT）

### 5.8 `Glossary`（术语表）
- 用户维护「英文术语 → 指定中文」键值表
- 翻译时把命中的术语作为约束注入提示词，保证全文译法一致

### 5.9 `InplaceRenderer`（原位替换渲染）
- 取每个文本块坐标与字号 → 白矩形覆盖（redact）原英文 → 原位排入中文，必要时缩字号/换行适配
- 图、表、公式不动，仅替换正文文本块
- ⚠️ 局限：中文偏长易溢出、字体差异、复杂排版可能错位——与双栏对照并存，按场景切换

### 5.10 `Cache`（缓存）
- SQLite，key = `hash(原文 + 目标语言 + 引擎/模型名)`；换模型/引擎算不同 key

### 5.11 `Vocabulary`（生词本）
- 收藏单词存 SQLite：单词、音标、释义、来源 PDF、时间
- 生词本面板可浏览/删除；**导出 Anki**（生成 `.txt`/`.csv` 或 `.apkg`，正面=单词、背面=音标+释义+例句）

### 5.12 `Settings`（配置）
- 管理各引擎 key（存 keyring）、当前引擎+模型、主题、自定义提示词、术语表、并发数
- 显示「当前缓存大小：X MB」+【清理缓存】按钮

## 6. 数据流

**划词（单词）**：选中 → 按键/图标触发 → 判定单词 → 查 ECDICT → 显示 → 异步补搭配/例句 → (可选)🔊 朗读 / ➕ 存生词本

**划词（短语）**：选中 → 触发 → 判定短语 → 查缓存 → 命中显示 / 未命中调引擎(流式) → 显示 → 存缓存

**整页/整篇**：`TextPreprocessor` 清洗重排 → 逐段查缓存 → 未命中交 `TranslateQueue`（限流+重试+进度，整篇先估费确认）→ 双栏排版 或 `InplaceRenderer` 原位写入 → 存缓存

## 7. 文件 / 数据位置（Windows）

```
配置(主题/提示词/术语表/并发):   %APPDATA%\PDFTranslator\config.json
API key:                       系统 keyring（不明文存盘）
翻译缓存(SQLite):               %LOCALAPPDATA%\PDFTranslator\cache.db
生词本(SQLite):                 %LOCALAPPDATA%\PDFTranslator\vocab.db
离线词典(只读，随程序自带):       <程序目录>\data\ecdict_lite.db
```

- 【清理缓存】只清空 `cache.db`，**不动词典与生词本**

## 8. 主题系统

- 颜色抽成 QSS 变量，每主题一份配色
- 内置：浅奶白(默认)、暗色现代、浅色清爽、深褐护眼、纯白；可复制扩展

## 9. 错误处理

- API key 未填 → 引导去设置
- 网络/限流/超时 → 自动退避重试；最终失败显示错误 + 手动重试按钮，绝不崩溃
- 整页某段失败 → 仅标记该段，不影响其他段
- 扫描件/无文字层 → 明确提示，不报错崩溃
- 原位替换排版异常 → 不影响双栏视图，可切回
